1、方法：一段具有独立功能的代码块，不调用就不执行

### 优点
① 可以将挤在一起的臃肿代码，按照功能进行分类管理，提高维护性  
② 提高代码的复用性

### 定义格式
```plain
public static void 方法名( ) {
    方法体
}
```

### 调用格式
方法名( )

### 方法注意事项
① 方法与方法之间是平级关系，允许嵌套定义。  
② 方法不调用就不执行  
③ 方法的定义顺序和执行顺序无关

## 5 方法的调用流程
① 方法没有被调用的时候，在方法区中的字节码文件中存放  
② 方法被调用的时候，需要进入到栈内存中运行

+ 方法区：字节码文件加载时进入的内存
+ 栈内存：方法运行时入栈，方法运行结束出栈

## 带参数的方法（黑马）
形参：全称形式参数，是指在定义方法时，所声明的参数  
实参：全称实际参数，调用方法时，所传入的参数

### 代码示例
```java
public class MethodDemo2 {
    public static void main(String[] args){
        getMax(10,20);
    }
    public static void getMax(int num1,int num2) {
        int max=num1>num2?num1:num2;
        System.out.println(max);
    }
}
```

**作用**：提高方法的灵活性

## 7 带返回值的方法
方法产生的数据，其他方法想要使用的话，需要定义带返回值的方法

### 代码示例
```java
public static void methodA(int a,int b){
    System.out.println(sum2);
}
public static void methodB(){}

public static int methodA (int a,int b){}
public static void method(){
    System.out.println(sum2=0);
    int sum = methodA(10, 20);
}
```

### 定义格式
```plain
public static 数据类型 方法名(数据类型 变量) {
    return 数据值;
}
```

### 调用格式
数据类型 变量名 = 方法名(参数);

## 8 方法通用定义格式
```plain
public static 返回值类型 方法名(参数) {
    方法体;
    return 数据;
}
```

### 定义方法的两个明确
1. 明确参数：主要是明确参数的类型和数量
2. 明确返回值类型：主要是明确方法操作完毕之后是否有结果数据
    - 如果有，写对应的数据类型
    - 如果没有，写void

### 调用方法的规则
+ void类型的方法，直接调用即可
+ 非void类型的方法，推荐用变量接收调用

## 9 方法注意事项（黑马程序员）
1. 方法不调用就不执行
2. 方法与方法之间是平级关系，不能嵌套定义
3. 方法的编写顺序和执行顺序无关
4. 方法的返回值类型为void，表示该方法没有返回值，没有返回值的方法可以省略return语句不写
5. 如果要编写return，后面不能跟具体的数据
6. return语句下面，不能编写代码，因为永远执行不到，属于无效的代码

## 10 方法重载
在同一个类中，定义了多个同名的方法。  
**简单记**：同一个类中，方法名相同，参数不同的方法。  
**参数不同**：个数不同、类型不同、顺序不同。  
**注意**：识别方法之间是否是重载关系，只看方法名和参数，跟返回值无关。
