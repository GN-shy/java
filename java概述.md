# Java 学习笔记
## 一、Java 概述
### 1. 程序定义
程序是计算机执行某些操作或**解决实际问题**而编写的一系列有序指令的集合。

### 2. Java 起源
+ 1995 年发布第一个版本
+ 创始人：James Gosling
+ 由太阳微系统公司（Sun Microsystems）负责开发

### 3. Java 技术体系平台
| 平台名称 | 说明 |
| --- | --- |
| Java SE（标准版） | 支持面向桌面级应用（如 Windows 平台的应用程序），提供完整的 Java 核心 API 工具 |
| Java EE（企业版） | 为开发企业环境下的应用程序提供一套解决方案 |
| Java ME（微型版） | 支持 Java 程序运行在移动终端（如手机、PDA 等），对 Java API 进行精简，并加入针对移动设备的特性 |


### 4. Java 核心特点
1. **面向对象（OOP）**
2. **健壮性**：强类型机制、完善的异常处理、自动垃圾回收等
3. **跨平台性**：一个编译好的 `.class` 字节码文件可在多个操作系统下运行（依赖 JVM）
4. **解释型**：属于解释性语言（同类语言：JavaScript、PHP），区别于编译性语言（如 C++）

### 5. JVM 核心机制
+ **定义**：JVM（Java 虚拟机）是一个虚拟的计算机，具有指令集并使用不同的内存区域，负责执行指令、管理内存
+ **核心特性**：
    1. 不同操作系统有对应的虚拟机（如 Windows 版 JVM、Linux 版 JVM）
    2. 屏蔽底层运行平台的差别，实现 **“一次编译，到处运行”**
+ **执行流程**：`Test.java`（源文件）→ 编译 → `Test.class`（字节码文件）→ 运行 → 输出结果

## 二、Java 程序编译与运行
### 1. 编译步骤
1. 准备 Java 源文件（后缀 `.java`）
2. 在源文件目录下通过编译器（javac.exe）对源文件编译（例：编译 `Hello.java`）
3. 若程序无错误，当前目录会生成对应的 `.class` 字节码文件（无错误提示）

### 2. 运行步骤
1. 找到编译生成的 `.class` 字节码文件
2. 通过运行工具（java.exe）执行字节码文件（例：运行 `Hello.class`，命令：`java Hello`）

### 3. 开发注意事项
+ 修改源文件后，必须**重新编译**生成新的 `.class` 字节码文件，再运行才能生效

## 三、Java 基础语法规范
### 1. 源文件与类规则
1. 源文件以 `.java` 为扩展名
2. 源文件的基本组成部分是类（class）
3. 一个源文件中**最多只能有一个 public 类**，其他类的个数不限
4. 若源文件包含 public 类，则**文件名必须与该类名完全一致**（大小写敏感）
5. 可将 main 方法写在非 public 类中，运行时指定该非 public 类即可（入口方法为该类的 main 方法）

### 2. 执行入口（main 方法）
+ 固定书写格式：`public static void main(String[] args) { ... }`

### 3. 语法细节
1. Java 语言**严格区分大小写**（例：`Hello` 与 `hello` 是不同类名）
2. 大括号 `{}` 成对出现，缺一不可（建议先写 `{}` 再填充代码）
3. 方法由多条语句构成，**每个语句以分号 **`;`** 结束**（不可省略）

### 4. Java 转义字符
| 转义字符 | 功能 |
| --- | --- |
| `\t` | 一个制表符，实现对齐功能 |
| `\n` | 换行符 |
| `\\` | 输出一个反斜杠 `\` |
| `\"` | 输出一个双引号 `"` |
| `\'` | 输出一个单引号 `'` |
| `\r` | 回车符 |


## 四、初学 Java 易犯错误及解决方法
| 错误类型 | 解决方法 |
| --- | --- |
| 找不到文件 | 1. 检查源文件名是否存在或拼写错误；2. 确认当前执行命令的路径正确 |
| 主类名和文件名不一致 | 声明为 public 的主类，其类名必须与文件名一致（大小写匹配），否则编译失败 |
| 缺少分号 | 编译会报错，根据错误提示的行号，到源代码对应位置补充分号 |


## 五、Java 注释
### 1. 注释的作用
+ 注解、说明程序代码，提高代码可读性
+ 是程序员的良好编程习惯：先通过注释整理思路，再用代码实现
+ 注释内容不会被 JVM 解释执行

### 2. 注释类型及格式
| 注释类型 | 基本格式 | 说明 |
| --- | --- | --- |
| 单行注释 | `// 注释文字` | 适用于简短注释，仅作用于当前行 |
| 多行注释 | `/* 注释文字 */` | 适用于多行注释，**不允许嵌套多行注释** |
| 文档注释 | `/**** 注释内容 ****/` | 注释内容可被 JDK 的 javadoc 工具解析，生成网页版程序说明文档，通常写在类、方法上方 |


### 3. 文档注释相关
#### （1）生成文档命令
```bash
javadoc -d 文件夹名 -xx -yy Demo3.java
```

+ `-d 文件夹名`：指定生成文档的输出目录

#### （2）常用 javadoc 标签
| 标签 | 功能 |
| --- | --- |
| `@author` | 标注类的作者 |
| `@version` | 指定类的版本 |
| `@deprecated` | 指明过期的类或成员 |
| `@docRoot` | 指明当前文档的根路径 |
| `@exception` | 标注类抛出的异常（与 `@throws` 功能相同） |
| `@inheritDoc` | 继承直接父类的注释 |
| `@link` | 插入到另一个主题的链接 |
| `@linkplain` | 插入到另一个主题的链接（显示plain文本） |
| `@param parameter-name explanation` | 说明方法的参数 |
| `@return explanation` | 说明方法的返回值类型 |
| `@see anchor` | 指定到另一个主题的链接 |
| `@serial` | 说明序列化属性 |
| `@serialData description` | 说明通过 `writeObject()` 和 `writeExternal()` 方法写入的数据 |
| `@serialField name type description` | 说明 `ObjectStreamField` 组件 |
| `@since release` | 标记引入特定变化的版本 |
| `@throws` | 标注方法抛出的异常（与 `@exception` 相同） |
| `{@value}` | 显示常量的值（常量必须是 static 属性） |


## 六、Java 代码规范
1. 类、方法的注释需以 **javadoc 文档注释** 格式编写
2. 非 javadoc 注释（单行/多行注释）：面向代码维护者，着重说明“为什么这么写”“如何修改”“注意事项”等
3. 缩进：使用 `Tab` 键实现缩进（整体右移），`Shift+Tab` 实现整体左移
4. 空格规范：运算符和 `=` 两边各加一个空格（例：`2 + 4 * 5 + 345 - 89`）
5. 编码格式：源文件使用 UTF-8 编码
6. 行宽限制：一行代码宽度不超过 80 字符
7. 代码风格：支持次行风格和行尾风格（例：次行风格如下）

```java
public ArrayList(int initialCapacity) {
    if (initialCapacity > 0) {
        // 代码逻辑
    }
}
```

3. JDK、JRE、JVM 的关系 [Homework03.java]  
问题：简述 JDK、JRE、JVM 的定义及三者之间的关系  
答案：  
（1）核心定义  
JVM（Java 虚拟机）：虚拟的计算机，具有指令集和独立内存区域，负责执行 Java 字节码；不同操作系统有对应的 JVM，屏蔽底层平台差异，实现 “一次编译，到处运行”。  
JRE（Java 运行时环境）：Java 程序运行所需的环境，包含 JVM 和 Java 核心类库（如 java.lang 包），是运行 Java 程序的最小依赖。  
JDK（Java 开发工具包）：Java 开发所需的工具集合，包含 JRE、编译器（javac.exe）、运行工具（java.exe）、文档生成工具（javadoc.exe）等。  
（2）三者关系  
包含关系：JDK ⊇ JRE ⊇ JVM（JDK 包含 JRE，JRE 包含 JVM）  
开发与运行的关联：开发时需安装 JDK（提供编译、文档生成等工具）；仅运行 Java 程序时，安装 JRE 即可（提供运行环境）；JVM 是跨平台的核心基础。
4. 环境变量 path 配置及其作用 [Homework04.java]  
问题：简述 Java 环境变量 path 的配置方式及核心作用  
答案：  
（1）核心作用  
让操作系统在任意目录下都能识别 javac（编译器）、java（运行工具）等 JDK 命令，无需每次执行命令时切换到 JDK 的 bin 目录。  
例如：配置后可在 D 盘目录直接执行 javac Homework01.java，无需切换到 C:\Program Files\Java\jdkXXX\bin 目录。  
（2）关键配置逻辑（以 Windows 系统为例）  
找到 JDK 安装目录下的 bin 文件夹路径（例：C:\Program Files\Java\jdk1.8.0_301\bin）。  
打开系统环境变量设置，在 “系统变量” 的 Path 变量中添加上述 bin 路径。  
配置完成后，重启命令行工具（CMD）即可生效。
5. Java 编写步骤 [Homework05.java]  
问题：简述 Java 程序从编写到运行的完整步骤  
答案：  
编写源文件：使用文本编辑器（如记事本、IDEA）创建后缀为 .java 的源文件，文件中包含类（class）和 main 方法（执行入口），遵循 “public 类名与文件名一致” 的规则。  
编译源文件：  
打开命令行，切换到源文件所在目录。  
执行编译命令：javac 源文件名.java（例：javac Homework01.java）。  
若程序无语法错误，会在当前目录生成后缀为 .class 的字节码文件（JVM 可识别）。  
运行程序：  
执行运行命令：java 类名（无需加 .class 后缀，例：java Homework01）。  
JVM 加载并执行字节码文件，控制台输出运行结果。  
修改后重新编译运行：若修改了源文件，需重复步骤 2 和 3，生成新的字节码文件后再运行，否则运行结果不变。
6. Java 编写 7 个规范 [Homework06.java]  
问题：列出 Java 编写的 7 个核心规范  
答案：  
类和方法的注释需使用 javadoc 文档注释 格式（/** ... _/），便于生成程序说明文档。  
__非 javadoc 注释（单行 //、多行 /_ ... _/）：面向代码维护者，着重说明 “为什么这么写”“修改注意事项” 等逻辑。  
__缩进规范：使用 Tab 键实现代码缩进（整体右移），Shift+Tab 实现整体左移，保证代码结构清晰。  
__空格规范：运算符（+、-、_、/ 等）和赋值符号（=）两边各加一个空格（例：2 + 4 * 5 而非 2+4*5）。  
编码规范：源文件统一使用 UTF-8 编码，避免中文乱码。  
行宽限制：一行代码的字符数不超过 80 个，超出部分换行，提高可读性。  
代码风格：采用 “次行风格” 或 “行尾风格” 编写代码（例：大括号 { 单独占一行或跟在语句末尾），保持风格统一。
